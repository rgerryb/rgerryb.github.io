8 Tren Web Dev 2018

1. Pahami Dasar-dasarnya

Tunggu apa? Saya mulai dengan menguraikan teknologi baru yang mengagumkan yang kita miliki dan sekarang kita terjun ke … dasar-dasarnya?
Ya, karena ini akan tetap menjadi hal terpenting yang harus Anda ketahui sebagai pengembang web pada tahun 2018! Kedengarannya sepele tapi karena kita memiliki lebih banyak teknologi untuk pengembangan web, penting untuk memahami apa yang ada di dalamnya.
Komponen web Itu hanya javascript! PWA? Juga JavaScript. Kerangka CSS? Coba tebak - mereka hanya menggunakan CSS biasa.
Anda tidak perlu menguasai semua teknologi ini - setiap pengembang memiliki area pilihan. Tapi Anda perlu tahu bagaimana HTML, CSS dan JavaScript bekerja dan bagaimana mereka bekerja sama. Tidak ada jalan lain.

2. Menyelam ke Node.js

Node.js telah populer selama beberapa tahun terakhir ini. Itu tidak akan berubah pada 2018, sebaliknya yang terjadi.
Tentu saja masih ada ruang untuk PHP, Ruby on Rails dll Tapi terutama dengan Serverless Web Apps yang sedang naik daun, Node.js adalah bahasa sisi server yang harus Anda lihat. Dan Anda tahu dasar-dasarnya (JavaScript!) Anyways, bukan? Node.js hanya JavaScript di server jadi mengapa tidak menyelam lebih dalam ke dalamnya dan menjadi pengembang frontend dan backend?
Selain itu, tidak ada satu keuntungan besar yang bisa membuat Node.js benar-benar lebih baik daripada PHP dll. Ini adalah bahasa yang menawarkan kinerja hebat. Ini menampilkan ekosistem yang dinamis dan aktif dengan banyak paket pihak ketiga yang dapat Anda pasang ke aplikasi Anda. Tapi alasan # 1 mengapa sangat populer adalah JavaScript menjadi lebih dan lebih penting. Dan itu akan terjadi pada 2018 juga.

3. Jelajahi Reakta, Sudut dan / atau Vue.js (Kerangka Pengarah)

Apakah saya sudah menyebutkan pentingnya JavaScript?
Kami akan banyak menjumpai artikel ini - yang menurut saya menggarisbawahi pentingnya hal tersebut.
Inilah salah satu bagian dari pengembangan web yang memancar: Kerangka kerja frontend seperti Angular, React atau Vue.js didukung sepenuhnya oleh (sisi klien) JavaScript.
Kerangka kerja ini (oh - React adalah perpustakaan, tolong jangan nyalakan saya) memungkinkan Anda menciptakan pengalaman pengguna yang sangat menarik di aplikasi web Anda karena semuanya dikontrol melalui JavaScript. Itu memungkinkan Anda untuk membuat ulang bagian halaman (atau keseluruhan halaman) setelah dimuat. Ini menghemat pemuatan ulang halaman yang tidak perlu dan meminta siklus tanggapan <-> dengan server. Pengalaman pengguna aplikasi seluler seperti aplikasi seluler adalah hasilnya - dan kami semua menginginkan pengalaman pengguna yang hebat, bukan?
Tepatnya, jika Anda merender ulang keseluruhan layar dan karenanya membuat ilusi memuat halaman yang berbeda, Anda membuat Aplikasi Halaman Tunggal (SPA) yang disebut satu kali (hanya satu halaman (index.html) yang dimuat dan sepertinya aplikasi web menghosting banyak halaman.
Semakin banyak aplikasi web dibuat sebagai SPA karena itu benar-benar menghadirkan perasaan seperti aplikasi seluler ke layar pengguna Anda. Tetapi bahkan jika Anda hanya mengendalikan sebagian halaman yang diberikan oleh server Anda dengan kerangka kerja ini (kecuali Sudut, itu benar-benar hanya bagus untuk menciptakan SPA), Anda dapat meningkatkan pengalaman pengguna karena Anda dapat menerapkan elemen dinamis seperti kameramen atau akordeon gambar. .
Anda bisa melakukan ini di masa lalu, dengan jQuery misalnya, tapi dengan React, Angular and Vue, itu menjadi lebih mudah dan lebih kuat.
Satu tren yang relatif baru yang mungkin akan mempercepat pada tahun 2018 adalah rendering pra-rendering atau server-side dari SPA. Itu berarti aplikasi Anda masih dikendalikan oleh JavaScript dan berjalan di browser namun Anda juga melayani versi pra-render aplikasi pada muatan pertama. Ini meningkatkan SEO (karena perayap melihat apa yang pengguna lihat, tanpa harus menunggu beberapa konten yang dimuat secara asinkron) dan dapat mempercepat aplikasi Anda.
Anda mungkin ingin melihat proyek seperti Next.js for React, Angular Universal for Angular atau Nuxt.js untuk Vue.js - mereka semua fokus untuk mempermudah ini.
Pengguna mungkin terus menyukai aplikasi web yang menarik, sehingga kerangka kerja frontend seperti Angular, React atau Vue.js akan tetap penting di tahun 2018. Jika Anda datang mencari pemenang, saya harus mengecewakan Anda sekalipun.
Ketiga kerangka kerja itu terlihat bagus saat ini. Anda dapat menemukan perbandingan (sangat berpendirian!) Di sini tapi akhirnya, saya sarankan menyelam ke semuanya (paling tidak sedikit) dan kemudian memilih favorit Anda.

4. Komponen Web & Kompiler Komponen

Tren # 1 sampai # 3 mungkin sudah tentang hal-hal yang sudah Anda miliki baru. Komponen Web (dan alat yang mengkompilasi Komponen Web seperti Stensil kurang dikenal dan hyped.
Gagasan di balik komponen web itu sederhana: Mengapa kita harus menggunakan perpustakaan seperti React untuk membangun komponen yang hanya berfungsi di perpustakaan itu dan tidak hanya membangun komponen ini dengan vanilla JavaScript dan menggunakannya juga di luar kerangka kerja?
Sampai akhir 2017, dukungan browser adalah masalah yang lebih besar namun pada awal 2018, ini terlihat lumayan. Empat API Web yang membentuk komponen web - Unsur Khusus, DOM Shadow, Template HTML dan Impor HTML - didukung oleh semua browser utama.
Terutama dua elemen pertama - Custom Elements dan Shadow DOM - adalah API Web yang sangat menarik.
Dengan elemen khusus, Anda dapat melakukan seperti apa rasanya: Buat elemen HTML Anda sendiri (misalnya ). Unsur-unsur ini berisi semua kode yang dibutuhkan untuk membawa sesuatu ke layar - tidak ada kerangka kerja yang dibutuhkan! Dan bahkan lebih baik dari itu: Anda dapat menggunakan elemen ini dalam hubungannya dengan kerangka kerja seperti Sudut, jika Anda memerlukan beberapa fungsi kerangka lainnya (yaitu di luar kemampuannya untuk menciptakan komponen yang dapat digunakan kembali).
Jenis DOM Bayangan dibangun di atasnya dan memungkinkan Anda juga menampilkan beberapa gaya CSS tertentu ke elemen khusus Anda. Ini memberi Anda elemen HTML yang benar-benar dapat digunakan kembali.
Template HTML memungkinkan Anda untuk menentukan kode HTML (menggunakan elemen ‘normal’ atau elemen khusus lainnya) di dalam elemen khusus Anda. Impor HTML akan memungkinkan Anda mengimpor file HTML ke dalam file HTML, standar ini sangat berat, namun sebenarnya tidak diperlukan karena kami biasanya menggunakan alat dan bundling seperti anyel anyaman Webpack.
Membangun komponen web / elemen kustom secara manual bisa menjadi sangat rumit seperti ditunjukkan contoh ini.
Karena itulah komponen-kompiler seperti StencilJS terlihat sangat menjanjikan. Elemen Sudut akan menjadi proyek lain yang memungkinkan Anda menggunakan sintaks yang lebih bagus untuk kemudian secara otomatis mengkompilasinya ke komponen web asli.
Ide inti di balik proyek ini adalah memberi Anda sintaks yang bagus dan banyak pembantu - Stensil menggunakan fitur TypeScript, JSX dan Angular, Elemen Sudut menggunakan Sudut - untuk membuat komponen web kustom. Kode Anda kemudian akan dikompilasi ke vanilla JavaScript, sehingga Anda dapat menggunakan komponen yang dibuat di halaman web manapun.
Anda tidak perlu menggunakan kerangka kerja tertentu atau semacamnya.
Kita dapat benar-benar bersemangat tentang komponen web karena menggunakannya memberi kita banyak kemungkinan untuk mengurangi ukuran bundel kita (Anda mungkin bisa membuat halaman web tanpa kerangka kerja apapun!). Selain itu, elemen khusus ini sangat dapat digunakan kembali di seluruh proyek. Kedengarannya seperti mimpi bagiku.

5.Gunakan Generator Halaman Statis

Performa menang, di tahun 2018 lebih dari sebelumnya.
Kami harus membuat aplikasi web kami untuk perangkat yang lebih banyak dan lebih berbeda, yang menjangkau laman web kami dari semua jenis tempat dengan segala jenis konektivitas. Pengiriman banyak JavaScript yang hanya mulai memuat konten kami setelah diunduh menjadi kurang dan kurang pilihan.
Selain itu, SEO jelas penting namun crawler mesin pencari tidak mengerti konten yang dimuat secara asynchronous (yaitu konten yang hanya dimuat dan renderer SETELAH pemuatan halaman pertama karena diambil dari beberapa server). Dan dengan semakin banyak aplikasi yang dibuat dengan cara tanpa server, kami mungkin juga tidak tertarik untuk merender sisi server laman kami.
Bagaimana semua hal ini sesuai? Bagaimana kita menyajikan versi halaman web yang telah ditentukan sebelumnya tanpa secara dinamis melakukan pra-render di beberapa server (seperti yang kita lakukan selama 20 tahun terakhir)?
Jawabannya adalah Generator Halaman Statis seperti Jekyll, Hugo atau proyek seperti Nuxt.js untuk Vue.js (ini terinspirasi oleh Next.js for React). Semua alat dan proyek ini memungkinkan kita melakukan satu hal: Buatlah halaman web yang kemudian kita buat di server lokal kita atau beberapa server build. Kami kemudian menyebarkan sekumpulan file statis - HTML, CSS dan JavaScript. Selesai!
Oke, Nuxt.js dan Next.js tidak berfokus pada hal itu, Anda juga bisa menggunakannya untuk merender SPA secara dinamis di server. Tapi tetap saja, fungsionalitas pembuatan halaman statis juga disertakan di sana.
Keuntungan besar menggunakan alat tersebut adalah kami bisa mengirimkan halaman yang telah dirender sementara masih bisa mengikuti pendekatan tanpa server. Kami akan menjadi orang yang optimis dan optimis dengan menghemat biaya!
Satu hal yang perlu diingat: Jika Anda membangun halaman yang menampilkan 10 dari 1000 halaman, pra-menghasilkan semuanya, mungkin beberapa kali per hari, bisa menjadi masalah.
Itulah kelemahannya: Jika Anda mengubah sesuatu sekitar satu halaman atau data yang termasuk dalam satu halaman, Anda harus mengembalikannya kembali (karena alatnya tidak tahu apa yang Anda ubah). Yang sedang berkata, penting untuk menyoroti bahwa hal tersebut menghasilkan menyala-cepat jadi ini mungkin tidak akan pernah menjadi masalah nyata bagi Anda.
Selain itu, Anda dapat menyempurnakan halaman yang Anda berikan dengan juga menyertakan beberapa fungsi pembaruan asinkron - yaitu kode yang memeriksa versi terbaru SETELAH halaman Anda telah dimuatkan ke browser pengguna Anda.

6.Aplikasi Web Progresif (PWA)

Progressive Web Apps (PWA) adalah istilah yang merangkum beberapa teknologi yang memungkinkan kami membuat aplikasi web yang berperilaku (dan terasa) sangat mirip dengan aplikasi seluler asli.
Itu sebagian besar mencakup fungsi offline (misalnya, bagian dari aplikasi web yang berfungsi tanpa sambungan aktif - setelah aplikasi web dimuat setidaknya satu kali), kemampuan terinstal (misalnya, aplikasi web dapat ditambahkan ke layar utama perangkat) dan penggunaan perangkat asli fitur (seperti mengakses kamera perangkat).
Oke, itu bagus tapi kenapa kita tidak membangun aplikasi asli saja?
Karena ini memiliki dua kelemahan utama: Fakta bahwa Anda harus belajar dua bahasa (Java / Android dan Swift / ObjectiveC) dan jangkauan Anda sangat terbatas. Membangun aplikasi seluler selalu terdengar menyenangkan dan mengagumkan sampai Anda melepaskannya dan Anda melihat bahwa tidak ada yang mendownloadnya.
Pengguna rata-rata memasang aplikasi baru nol (ya 0) setiap bulan!
Itu tidak terlalu banyak, bukan? Di sisi lain, orang mencari barang di Google setiap saat. Dan mereka menemukan: aplikasi web Anda. Bingo, itulah mengapa ide bagus untuk membawa yang terbaik dari kedua dunia - persis seperti yang dilakukan PWA.
Itu saja akan sangat mengagumkan tapi bagian ‘progresif’ yang membuat teknologi ini luar biasa. Anda tidak perlu pergi semua-dalam fitur PWA, Anda malah bisa menerapkan apa yang paling Anda butuhkan. Ingin aplikasi web yang dapat diinstal? Fokus pada itu, semudah itu.
Jadi, teknologi apa yang sedang saya bicarakan? Sebagian besar Pekerja Layanan dan Aplikasi Web Manifest (file manifest.json).
Pekerja Layanan mengizinkan Anda untuk mengendalikan cache browser khusus, sehingga memberi Anda kekuatan untuk melakukan pra-cache aset tertentu dan menentukan kapan harus dimuat dari cache dan dalam kasus mana jaringan harus digunakan.
File manifest.json di sisi lain memberikan informasi meta yang berguna tentang aplikasi Anda ke browser, yang memungkinkan aplikasi dipasang di layar awal.
Tentu saja ada cara lain untuk melakukannya dan Anda dapat menyelam ke Panduan Lengkap saya tentang PWA jika Anda ingin mempelajari lebih lanjut tentang teknologi mengagumkan ini.

7.Bangun Aplikasi Tanpa Server

Tanpa server Apa artinya itu? Bagaimana seharusnya halaman web tanpa server bekerja?
Jawabannya sederhana: tidak. Tapi istilah Serverless mengacu pada sesuatu yang berbeda.
Serverless berarti Anda tidak perlu khawatir mengelola dan memperbarui server, sehingga kapasitas server Anda meningkat drastis dan Anda tetap hanya membayar untuk apa yang Anda butuhkan. Tidak ada pengunjung yang tidak (atau hampir tidak) biaya. Semudah itu
Bagaimana cara kerjanya?
Anda hanya menulis kode Anda yang harus dijalankan sesuai permintaan (misalnya saat pengguna berlangganan newsletter Anda), mengunggahnya ke layanan tanpa server (misalnya AWS Lambda), mungkin menambahkan beberapa layanan tambahan (misalnya, AWS API Gateway untuk dengan mudah membangun REST tanpa server API) dan Anda menerapkan aset statis Anda ke beberapa host statis seperti AWS S3.
Itu saja, aplikasi / kode Anda sekarang hanya berjalan bila diperlukan, yaitu saat pengguna berinteraksi dengannya, dan karena itu Anda tidak membayar waktu idle apapun. Selain itu, Anda tidak perlu khawatir mengkonfigurasi server Anda dengan benar karena tidak ada server.
Bagi saya, ini terdengar cukup bagus, jadi tidak mengherankan jika serverless adalah topik hangat dan akan tetap panas untuk tahun 2018.
Sementara banyak pekerjaan dasar dilakukan selama tahun-tahun terakhir, sekarang kami mendapatkan lebih banyak alat untuk membangun aplikasi serverless yang luar biasa (misalnya Kerangka Serverless) dan dengan itu DevOps menjadi lebih mudah / lebih baik untuk Serverless juga. Perlu juga dicatat bahwa sementara Amazon Web Services (AWS) adalah inovator yang kuat di bidang itu, sekarang kami mendapatkan lebih banyak lagi perusahaan yang menawarkan layanan tanpa server - baik Microsoft Azure maupun Google Cloud memiliki layanan tanpa server.
Jadi secara keseluruhan, pengembangan aplikasi serverless menjadi lebih menyenangkan, pengujian menjadi lebih mudah dan semakin banyak sumber daya untuk belajar bagaimana membangun aplikasi web serverless agar tersedia. Saya juga punya kursus tentang itu.

8.Kecerdasan Buatan dan Bot

Oke, Anda pasti sudah sedih jika tidak disertakan, bukan?
Mungkin tidak ada topik yang lebih disukai daripada Inteligensi Buatan (AI) dan bot akhir-akhir ini.
Tapi sejujurnya: Ini adalah topik hangat dan AI akan menjadi lebih dan lebih penting selama tahun depan - termasuk tahun 2018.
AI TIDAK terbatas pada mobil self-driving. Pikirkan tentang chat bots di laman Anda yang membantu pengguna memahami konten Anda. Pertimbangkan alat pengenalan gambar dan pengoptimalan. Dan seterusnya.
Ada banyak use-cases untuk komponen AI-driven di halaman web. Kami hanya menyelam ke era itu tapi tidak ada salahnya menjadi orang pertama yang menyelam ke hal-hal baru yang akan bertahan.
AI adalah bidang yang luas sekalipun. Anda bisa menyelam ke Python, Machine Learning, Big Data, Deep Learning dan membangun alat AI sendiri. Jika Anda menguasai semua itu, Anda akan memiliki masa emas di depan Anda;).
Namun, untuk penggunaan lainnya, kami dapat menyelami lebih banyak alat yang memungkinkan kami menerapkan layanan AI ke semua jenis aplikasi dan model bisnis tanpa persyaratan menjadi ahli AI.
Pertimbangkan alat seperti AWS Lex, layanan yang memungkinkan Anda membangun bots obrolan cerdas yang dapat Anda terapkan ke halaman Anda. Alat seperti ini membuat AI dapat diakses oleh kita semua dan memungkinkan Anda untuk berfokus pada ide dan logika bisnis Anda. Dan kita akan melihat banyak alat seperti ini muncul (dan menjadi lebih baik) di tahun 2018 - oleh karena itu ini adalah tren yang mungkin tidak ingin Anda lewatkan.
