Git - Panduan Sederhana

Hanya panduan sederhana untuk memulai dengan Git. Tanpa omong kosong ;)
Oleh Roger Dudler

Buat Repositori Baru

buat direktori baru, buka dan lakukan
git init
untuk membuat repositori git baru.

Checkout Repositori

buat salinan repositori lokal dengan menjalankan perintah
git clone / path / to / repository
bila menggunakan remote server, perintah anda akan
git clone username @ host: / path / to / repository

Alur Kerja

repositori lokal Anda terdiri dari tiga “pohon” yang dikelola oleh git.
yang pertama adalah Direktori Kerja Anda yang menyimpan file sebenarnya.
yang kedua adalah Index yang bertindak sebagai area pementasan dan akhirnya
HEAD` yang menunjuk pada komit terakhir yang telah Anda buat.

Tambahkan & Komit

Anda bisa mengajukan perubahan (tambahkan ke Index) menggunakan
git tambahkan <filename>
git tambahkan *
Ini adalah langkah pertama dalam alur kerja git dasar. Untuk benar-benar
melakukan perubahan ini gunakan
git commit -m "pesan komit"
Sekarang file tersebut berkomitmen pada HEAD, tapi tidak di repositori jauh
Anda.

Mendorong Perubahan

Perubahan Anda sekarang ada di bagian bawah salinan pekerjaan lokal Anda.
Untuk mengirim perubahan tersebut ke repositori jauh Anda, jalankan
git origin master
Ubah master menjadi cabang apa pun yang ingin Anda dorong untuk berubah.
Jika Anda belum mengkloning repositori yang ada dan ingin menghubungkan
repositori Anda ke server jauh, Anda perlu menambahkannya
git remote add origin <server>
Sekarang Anda dapat mendorong perubahan Anda ke server jarak jauh yang
dipilih.

Percabangan

Cabang digunakan untuk mengembangkan fitur yang terisolasi satu sama lain.
Cabang induk adalah cabang “default” saat Anda membuat repositori. Gunakan
cabang lain untuk pengembangan dan menggabungkan mereka kembali ke cabang
induk setelah selesai.
buat cabang baru bernama “feature_x” dan aktifkan dengannya
git checkout -b feature_x
kembali ke master
git checkout master
dan hapus cabang lagi
git branch -d feature_x
cabang tidak tersedia untuk orang lain kecuali jika Anda mendorong cabang
ke repositori jauh Anda
git push origin

Perbarui & Gabungkan

Untuk memperbarui repositori lokal Anda ke komit terbaru, jalankan
git pull
di direktori kerja Anda untuk mengambil dan menggabungkan perubahan jarak jauh.
Untuk menggabungkan cabang lain ke cabang aktif Anda (misalnya master), gunakan
git merge <branch>
Dalam kedua kasus ini git mencoba menggabungkan otomatis perubahan.
Sayangnya, hal ini tidak selalu memungkinkan dan berakibat pada konflik.
Anda bertanggung jawab untuk menggabungkan konflik tersebut secara manual
dengan mengedit file yang ditunjukkan oleh git. Setelah berubah, Anda perlu
menandai mereka sebagai penggabung
git add <filename>
Sebelum menggabungkan perubahan, Anda juga dapat melihat pratinjau dengan menggunakan
git diff <source_branch> <target_branch>

Penandaan

Sebaiknya buat tag untuk rilis perangkat lunak. Ini adalah konsep yang
dikenal, yang juga ada di SVN. Anda bisa membuat tag baru bernama 1.0.0
dengan mengeksekusi
git tag 1.0.0 1b2e1d63ff
1b2e1d63ff adalah singkatan dari 10 karakter pertama id komit yang ingin
Anda rujuk dengan tag Anda. Anda bisa mendapatkan komit id dengan melihat
…

Log

Dalam bentuknya yang paling sederhana, Anda bisa mempelajari sejarah
repositori dengan menggunakan .. git log
Anda bisa menambahkan banyak parameter agar log terlihat seperti yang Anda
inginkan. Hanya untuk melihat komit seorang penulis tertentu:
git log --author = bob
Untuk melihat log yang sangat dikompres dimana masing-masing komit adalah
satu baris:
git log --pretty = oneline
Atau mungkin Anda ingin melihat pohon seni ASCII dari semua cabang, dihiasi
dengan nama tag dan cabang:
git log --graph --oneline --decorate --all
Lihat saja file yang telah diubah:
git log - nama-nama
Ini hanya beberapa parameter yang mungkin Anda gunakan. Untuk informasi
lebih lanjut, lihat git log --help

Ganti Perubahan Lokal

Jika Anda melakukan sesuatu yang salah, yang pastinya tidak akan pernah
terjadi;), Anda dapat mengganti perubahan lokal menggunakan perintah
git checkout - <filename>
ini menggantikan perubahan di pohon kerja Anda dengan konten terakhir di
HEAD. Perubahan yang sudah ditambahkan ke indeks, serta file baru, akan
disimpan.
Jika Anda ingin menghapus semua perubahan dan komit lokal Anda, ambil
riwayat terbaru dari server dan arahkan cabang master lokal Anda seperti
ini.
git fetch origin
git reset --hard origin/master

Petunjuk Berguna

built-in git GUI
gitk
gunakan output git berwarna-warni
git config color.ui true
tampilkan log on hanya satu baris per komit
git config format.pretty oneline
gunakan penambahan secara interaktif
git add -i
